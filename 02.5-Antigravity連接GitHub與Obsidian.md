## 這個懶人包會幫你做什麼？

完成後，Antigravity 可以：

- 檢查 Git / GitHub CLI 是否已安裝
- 用網頁端登入 GitHub CLI
- 設定 Git 使用者資訊
- 建立測試 repo，驗證 Antigravity 可以 commit / push
- 找到既有 Obsidian vault
- 安裝 `@bitbonsai/mcpvault`
- 寫入 Antigravity MCP 設定 (`mcp_config.json`)
- 重啟後驗證 Antigravity 可以讀寫 Obsidian 筆記

# 階段一：設定 GitHub CLI

## 步驟一：檢查 Git 與 GitHub CLI

請 Antigravity 執行：

```powershell
git --version
gh --version
```

如果 Git 沒有安裝，Windows：

```powershell
winget install --id Git.Git --accept-source-agreements --accept-package-agreements
```

如果 GitHub CLI 沒有安裝，Windows：

```powershell
winget install --id GitHub.cli --accept-source-agreements --accept-package-agreements
```

安裝後，請完全重啟 Antigravity，再重新檢查。

---

## 步驟二：用網頁端登入 GitHub CLI

先檢查：

```powershell
gh auth status
```

如果尚未登入：

```powershell
gh auth login --web --git-protocol https
```

流程：

1. PowerShell 顯示一次性驗證碼。
2. 瀏覽器開啟 GitHub device login。
3. 若沒自動開啟，手動開：

```text
https://github.com/login/device
```

4. 輸入驗證碼並授權。
5. 回到 Antigravity，重新執行：

```powershell
gh auth status
```

成功時應看到：

```text
Logged in to github.com account <你的帳號>
Git operations protocol: https
Token scopes: ... repo ...
```

---

## 步驟三：設定 Git 使用者資訊

檢查：

```powershell
git config --global user.name
git config --global user.email
```

如果沒有設定，請詢問使用者姓名與 email：

```powershell
git config --global user.name "你的姓名"
git config --global user.email "你的email@example.com"
```

---

## 步驟四：建立測試 repo，驗證 commit / push

建立測試資料夾：

```powershell
New-Item -ItemType Directory -Path "$env:USERPROFILE\Documents\antigravity-github-test" -Force
Set-Location "$env:USERPROFILE\Documents\antigravity-github-test"
```

建立測試檔：

```powershell
@"
# Antigravity GitHub 測試

如果這個檔案成功出現在 GitHub，代表 Antigravity 可以 commit 與 push。
"@ | Set-Content -Encoding UTF8 -Path ".\README.md"
```

初始化並提交：

```powershell
git init
git add README.md
git commit -m "建立 Antigravity GitHub 測試"
```

建立 GitHub repo 並 push：

```powershell
gh repo create antigravity-github-test --private --source=. --push
```

驗證：

```powershell
gh repo view antigravity-github-test --web
```

---

## 步驟五：測試 repo 要保留還是刪除

測試成功後，請問使用者：

```text
GitHub 測試成功。這個 antigravity-github-test repo 要保留，還是刪除？
```

若要刪除 GitHub repo：

```powershell
gh repo delete antigravity-github-test --yes
```

若要刪除本機測試資料夾：

```powershell
$target = Join-Path $env:USERPROFILE "Documents\antigravity-github-test"
if ((Test-Path -LiteralPath $target) -and ((Resolve-Path $target).Path -eq (Join-Path $env:USERPROFILE "Documents\antigravity-github-test"))) {
    Remove-Item -LiteralPath $target -Recurse -Force
}
```

# 階段二：設定 Obsidian MCPVault

## 步驟六：找到使用者的 Obsidian vault

先問使用者：

```text
你的 Obsidian 筆記本放在哪裡？如果不知道，我可以幫你找。
```

常見位置：

| 同步方式 | 常見路徑 |
|----------|----------|
| OneDrive | `C:\Users\<你>\OneDrive\文件\<vault名稱>` |
| Google Drive | `G:\我的雲端硬碟\<vault名稱>` |
| Documents | `C:\Users\<你>\Documents\<vault名稱>` |
| Obsidian Sync | 本機任意資料夾，只要裡面有 `.obsidian` |

如果使用者不知道，Windows 可搜尋：

```powershell
$roots = @(
  "$env:USERPROFILE\OneDrive",
  "$env:USERPROFILE\Documents",
  "$env:USERPROFILE\Desktop",
  "G:\我的雲端硬碟",
  "G:\My Drive"
)

$roots |
  Where-Object { Test-Path $_ } |
  ForEach-Object {
    Get-ChildItem -Path $_ -Recurse -Directory -Force -ErrorAction SilentlyContinue |
      Where-Object { Test-Path (Join-Path $_.FullName ".obsidian") } |
      Select-Object FullName
  }
```

找到候選 vault 後，請使用者選主要 vault，不要自行猜。

確認條件：

- 資料夾存在
- 裡面有 `.obsidian`
- 使用者確認這是他的主要筆記本

把路徑記成：

```text
<VAULT_PATH>
```

---

## 步驟七：檢查 Node.js 與 npm

mcpvault 需要 Node.js / npm。

請執行：

```powershell
node --version
npm.cmd --version
```

如果沒有 Node.js，Windows：

```powershell
winget install --id OpenJS.NodeJS --accept-source-agreements --accept-package-agreements
```

安裝後重啟 Antigravity，再確認：

```powershell
node --version
npm.cmd --version
```

---

## 步驟八：安裝 mcpvault

Windows PowerShell 建議用 `npm.cmd`，避免 PowerShell 執行原則擋到 `npm.ps1`。

```powershell
npm.cmd install -g @bitbonsai/mcpvault
```

確認安裝位置：

```powershell
where.exe mcpvault
```

常見位置：

```text
C:\Users\<你>\AppData\Roaming\npm\mcpvault.cmd
```

把這個完整路徑記成：

```text
<MCPVAULT_CMD>
```

---

## 步驟九：寫入 Antigravity MCP 設定

Antigravity 設定檔在：

```text
C:\Users\<你>\.gemini\antigravity\mcp_config.json
```

請先備份：

```powershell
$config = "C:\Users\user\.gemini\antigravity\mcp_config.json"
$backup = "$config.bak-$(Get-Date -Format 'yyyyMMdd-HHmmss')"
Copy-Item -LiteralPath $config -Destination $backup -Force
```

在 `mcpServers` 區塊中加入 `obsidian` 的 JSON 設定。

Windows 範例：

```json
    "obsidian": {
      "command": "C:\\Users\\user\\AppData\\Roaming\\npm\\mcpvault.cmd",
      "args": ["G:\\我的雲端硬碟\\88-Obsidian"]
    }
```

> [!warning]
> Windows JSON 路徑要用雙反斜線 `\\` 進行跳脫。
> `args` 裡的 vault 路徑也要用 `\\`。

---

## 步驟十：手動測試 mcpvault

在重啟 Antigravity 前，先測 mcpvault 本身能不能啟動。

```powershell
'{"jsonrpc":"2.0","id":1,"method":"tools/list"}' |
  & "<MCPVAULT_CMD>" "<VAULT_PATH>"
```

如果成功，應該會看到工具清單，包含類似：

```text
read_note
write_note
search_notes
list_directory
get_vault_stats
```

若這一步失敗，先不要重啟 Antigravity，先排查路徑。

---

## 步驟十一：重啟 Antigravity 並驗證 Obsidian MCP

請使用者：

1. 完全關閉 Antigravity 視窗
2. 重新開啟
3. 測試讀寫與列出目錄

測試方式：

```text
請列出我的 Obsidian vault 根目錄資料夾。
```

再測寫入：

```text
請在我的 Obsidian 筆記本建立 `Antigravity 測試/連線測試.md`，內容寫入今天日期，並記錄「Antigravity 已成功連接 Obsidian MCP」。
```

## 踩坑筆記

| 狀況 | 原因 | 解法 |
|------|------|------|
| `git` 找不到 | Git 未安裝或 Antigravity 尚未重啟 | 安裝 Git 後重啟 Antigravity |
| `gh auth status` 顯示未登入 | GitHub CLI 尚未授權 | `gh auth login --web --git-protocol https` |
| `gh auth login --web` 等到逾時 | 需使用者到瀏覽器完成授權 | 告知使用者驗證碼，並至瀏覽器輸入進行授權 |
| `npm.ps1` 被擋 | PowerShell 執行原則 | 改用 `npm.cmd` |
| JSON 格式解析失敗 | 修改 `mcp_config.json` 時漏了逗號或雙引號 | 檢查 JSON 語法並補上逗號 |
| 反斜線失效 | Windows 路徑未跳脫 | 使用 `C:\\Users\\...` |
| 找不到 vault | 路徑錯或不是 Obsidian vault | 搜尋 `.obsidian` 並請使用者確認 |
