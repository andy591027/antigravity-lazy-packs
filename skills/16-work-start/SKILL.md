# 開工技能 (Startup Sync)

當使用者要求「開工」或「準備開始工作」時，AI Agent 應執行以下任務，幫助開發環境進入最佳狀態：

## 步驟一：遠端同步與代碼狀態檢查
1. **檢查 Git 狀態**：執行 `git status` 確認本地是否有未提交的變更。
2. **拉取遠端最新代碼**：若已關聯遠端 GitHub，執行 `git pull` 以確保本地代碼為最新版本，避免衝突。
3. **回報目前分支與狀態**：向使用者簡報目前所在的 Git 分支及乾淨程度。

## 步驟二：載入與喚醒第二大腦進度
1. **讀取 `AGENTS.md`**：了解該專案的核心規範與協作共識。
2. **讀取 `task.md` 或 `README.md`**：找出尚未完成、進行中、或最高優先級的 TODO 任務。
3. **讀取最新 Obsidian 工作筆記**（若有配置）：了解前一次收工時留下的遺留問題與備忘錄。

## 步驟三：自動建置與依賴檢查
* 檢查專案依賴是否需要更新（例如：若 `package.json` 有變動，提示或執行 `npm install`；若有 `requirements.txt` 則提示虛擬環境狀態）。

## 步驟四：開工激勵與任務總覽
向使用者回報：
1. ☀️ **早安/日安問候**。
2. 🔄 **代碼同步狀態**（例如：已是最新、或有變動）。
3. 🎯 **本日核心任務建議**：條列出 `task.md` 中優先級最高的前 3 個待辦事項。
4. **主動詢問**：「請問我們今天從哪一項任務開始著手？」
