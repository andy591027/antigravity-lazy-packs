# 收工技能 (Shutdown Sync)

當使用者要求「收工」、「結束工作」或「今天到這裡」時，AI Agent 應執行以下整理與歸檔任務：

## 步驟一：代碼品質與敏感資訊檢查
1. **執行測試/檢查**：若有配置測試，可執行測試確保無破壞性變更（例如 `npm test`）。
2. **敏感資訊過濾**：檢查是否有誤將 API Key、`.env` 或個人私密設定暴露在 Git 追蹤範圍內。

## 步驟二：自動提交進度 (Git Save)
1. **檢查變更**：執行 `git status` 與 `git diff --stat` 整理今日修改了哪些檔案。
2. **暫存並提交**：
   ```powershell
   git add .
   # 根據今日修改的核心模組，自動組裝出符合 Conventional Commits 規範的訊息
   git commit -m "feat/chore: auto save progress at session end"
   ```

## 步驟三：推送與備份至遠端
1. **推送代碼**：若有設定遠端 GitHub 儲存庫，執行 `git push` 將變更安全推送到雲端。
2. **組裝設定同步**（選擇性）：若有配置 chezmoi，執行同步以備份全域設定。

## 步驟四：更新工作進度（第二大腦）
1. **更新 `task.md`**：將今天已完成的任務標記為 `[x]`，進行中的標記為 `[/]`。
2. **產出收工備忘錄（Obsidian / Walkthrough）**：在 Obsidian 筆記或專案進度檔中，寫下今日完成亮點，以及**「下一次開工應繼續的起點」**，確保進度不中斷。

## 步驟五：收工匯報與溫馨告別
向使用者展示：
1. 🌙 **收工溫馨問候**。
2. 📝 **今日工作成果清單**（自動彙整 commit 變更與任務更新）。
3. 📌 **明日待續起點**（提醒下一次開工應首要著手的事項）。
4. 🔒 **安全回報**：已成功同步至 GitHub 遠端儲存庫。
